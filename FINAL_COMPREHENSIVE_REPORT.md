## 问题诊断与解决方案

### 问题 1: 语言切换功能不工作

**症状**
- 点击语言选择器时下拉菜单无法显示
- 语言切换按钮无响应

**根本原因**
- `language-switcher.js` 脚本虽然被加载，但事件监听器未正确附加
- 脚本在 HTML 末尾加载时，`DOMContentLoaded` 事件已经触发过
- 导致初始化函数未被执行

**解决方案**
- 在所有 HTML 文件中添加内联脚本，在 `<head>` 末尾初始化语言切换器
- 确保脚本在 DOM 完全加载后立即执行

**修改文件**
```
✅ /home/ubuntu/warkAI_website/index.html
✅ /home/ubuntu/warkAI_website/index-en.html
✅ /home/ubuntu/warkAI_website/index-ja.html
```

---

### 问题 2: Gundam 导航标签不可见

**症状**
- 机器人身体部位的导航标签完全隐藏
- 用户无法看到可点击的导航选项

**根本原因**
- CSS 中 `.part-label` 的默认 `opacity: 0` 隐藏了标签
- 标签只在悬停时显示（`:hover` 状态）
- 这是原始设计的一部分，但标签完全不可见

**解决方案**
- 移除之前添加的 `opacity: 1` 强制显示样式
- 恢复原始 CSS 行为：默认隐藏，悬停时显示并发光

**验证**
- ✅ 默认状态：标签隐藏
- ✅ 悬停状态：标签显示并发光
- ✅ 点击导航：正确跳转到对应页面
